Python 全栈开发学习笔记

记录从零开始构建 Web 应用的技术栈与实践心得。

01. Flask 框架基础

Flask 是一个微型的 Python Web 框架。相比于 Django，它更加轻量和灵活。在构建应用时，我们需要理解其核心概念：路由 (Routes)、模板渲染 (Jinja2) 以及 请求上下文 (Request Context)。

核心思想：Keep it simple, stupid. 只提供核心功能，其他功能通过扩展实现。

常用扩展库整理

  • Flask-WTF: 处理表单验证与 CSRF 保护。
  • Gunicorn: 生产环境下的 WSGI HTTP 服务器。

02. 数据库设计与 ORM

在数据持久化层面，选择了 PostgreSQL 作为主数据库。相较于 MySQL，Postgres 在处理复杂查询和 JSON 数据类型上有着天然的优势。

# 示例：使用 Python 连接数据库
import psycopg2

conn = psycopg2.connect(
    dbname="mydb",
    user="user",
    password="password",
    host="localhost"
)

在实际开发中，还需要考虑数据库的安全性，例如使用行级安全策略（Row Level Security, RLS）来隔离不同用户的数据访问权限。

03. Linux 服务器运维

从本地开发环境迁移到生产环境（阿里云 ECS），主要涉及环境搭建和反向代理配置。Nginx 作为高性能的反向代理服务器，主要负责静态资源托管和 SSL 证书的终结。

此外，为了服务器安全，配置 Fail2Ban 防止暴力破解 SSH 也是必不可少的步骤。
